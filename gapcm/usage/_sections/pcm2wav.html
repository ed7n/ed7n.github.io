A consumer PCM file should be headered to be conventionally usable. This can be done in many ways.

FFmpeg

You can pipe the decoder output to [<{{htmlAOut}}"https://ffmpeg.org" >FFmpeg] and save it as a WAVE file:

$ ./gamdec -o - lunar2/rp040.pcm | ffmpeg -hide_banner -f u8 -ar 16276 -ac 1 -i - -c:a pcm_u8 rp040.wav

For stereo, set -ac to 2. Details can be found in [<{{htmlAOut }}"https://trac.ffmpeg.org/wiki/audio%20types">their wiki].

SoX

If FFmpeg is too noisy, then you can do the same with [<{{htmlAOut}}"https://sox.sourceforge.net" >SoX]:

$ ./gamdec -o - lunar2/rp041.pcm | sox -b 8 -e un -r 16276 -t raw -c 1 - rp041.wav

For stereo, set -c to 2.

Tenacity

You can also import the decoder output files into [<{{htmlAOut}}"https://tenacityaudio.org" >Tenacity]: File→Import→Raw Data…, pick your file, set the appropriate values, then import away. From here, you can do whatever you want to.

The same goes for [<{{htmlAOut}}"https://www.audacityteam.org">Audacity].
