The --check action inspects your cuesheet for adherence to the original specification based from GoldenHawk Technology's [<{{ htmlAOut }} "https://web.archive.org/web/20070221154246/http://www.goldenhawk.com/download/cdrwin.pdf" >CDRWIN User's Guide] but with a few significant differences:

  • FILE statements may reside anywhere.
  • PERFORMER, SONGWRITER, and TITLE arguments may have zero or over 80 characters.
  • The first track number must be 1 for now (TRACK 01).

However, it does not catch duplication or misplacement errors, such as:

  • Multiple PERFORMERs in a TRACK, in which case QEdit keeps only the bottommost one.
  • A TITLE after an INDEX, which will be parsed and correctly placed on output.

It also checks for missing or invalid FILEs, such as directories, though it does not look into their contents.

The following shows the --check action in practice:

$ java -jar qedit.jar --check jcroot.cue
TRACK 01 INDEX 01: Time code is less than INDEX 00
TRACK 01 INDEX 00: FILE "jcroot.wav" does not exist
TRACK 03 INDEX 00: Time code is less than or equal to TRACK 02 INDEX 01
