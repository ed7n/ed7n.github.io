03/26/2019

As it was rushed, Development A was terrible--but it worked. It's different from the later releases in that it accepts as a directory input path to process cuesheets recursively. QEdit was described as "hopefully a CUE sheet editor."

03/27/2019

Development B introduced actions, starting with pregapToIndex00, its reverse counterpart, and print.

06/13/2019

Development C improved the cuesheet parser, brought more actions including --AdjustTime, and removed the directory recursion feature.

04/18/2020

Development D marked this project as the first to have its own section in the Online Portal, the first to integrate with my Java API, and the first to support REM statements. The cuesheet checker was matured to comply with most of the original specifications.
