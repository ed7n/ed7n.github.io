Division

Optionally, you can set the division algorithm and precision either within or outside of {{data["pnic"].body.title}}. See the script for names.

The Pad, Count, Shift (pcs) algorithm allows for any precision. It is not bound by Bash's limits, but a large precision like 9999 can make division unusable. The default precision should suffice, so change it only when required by your application. This algorithm is the default as we recommend using it.

The Padding Dividend (pdv) algorithm is still available, with which your precision setting does not apply as it is fixed. Note that this algorithm is very prone to rounding errors.
